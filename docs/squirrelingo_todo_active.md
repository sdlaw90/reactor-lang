### New feature notes (captured 2026-07-18)

- **87:** Answer-choice voice playback. **RIDES #37** — shares the TTS pipeline AND the review/delayed-quiz-mode gate; do NOT build before the #37 TTS-approach verdict. **Tap-to-play only** (no autoplay). Choice-audio controls appear **only when review/pause mode is on**; hidden in normal quiz flow. **Excluded from progress aggregates** until fully built (same rule as #37 question audio). Design note: voicing the options can quietly change what an item tests (reading → listening) or hint the answer — the mode gate + tap-only keeps it opt-in and out of the default flow. Cost: TTS scales with option count (WB already ~5×'s per-track chars) — budget the synth; not a blocker.

- **88:** Thematic learning as a **TAG/FILTER LAYER, not new categories** — this is what keeps it on the right side of the anti-forced-taxonomy principle and leaves the mix-and-match picker + mastery unfragmented (same integrity concern as #66/#77). Model: every item keeps ONE **home category** (mastery counts there, unchanged) plus **many-to-many theme tags**; the practice picker gains a **theme filter** that pulls every tagged item from any category; results still record to the home category → **no new theme mastery bar**. Cost = a content **TAGGING PASS** that should share a single tag model with #89's tense tags (plan once, reuse). **Ship a curated starter set (~10–12), NOT all ~40** — choice paralysis hits ADHD users hard; grow as content gets tagged. Starter: Greetings & intros · Numbers/Dates/Time · Directions · Shopping · Restaurant · Travel · Medical/Doctor · Small talk · Work/Office · Emotions. Future themes (grow later): airport/hotel/transit/driving/sightseeing/emergencies-abroad; groceries/cooking/dietary; pharmacy/body-symptoms/mental-health/fitness; IT/meetings/interviews/school/email-etiquette; family/dating/holidays/hobbies/texting/entertainment; banking/housing/gov-paperwork/post/legal; gaming/nature-animals/cars/sports-fandom. Overlap is expected & fine (e.g. pay/card → Shopping + Banking + Restaurant).

- **89:** Tense **"training-wheels" toggle**. Reframes the **rejected** "separate per-tense categories" (Business Plan v1.6 §5 — a forced taxonomy that fragments the mastery tracker + picker) around the real gap: **tense is invisible in questions** — we show the dictionary verb but not what tense a form is in or why, so learners can't build form→tense→meaning. Fix: **surface the tense + a one-line "why," ON by default, DISMISSABLE at advanced levels** (where the learner should recognize it unaided). Explanation-layer feature → content rides **#60/#69**; engine is small. Backed by the **per-item tense/grammatical-form tagging pass** (shares the tag model with #88). Note: for "choose the correct form" items, naming the tense can hand over the answer — that's the intended scaffolding, hence the dismissable toggle. **Cross-language caveat:** "grammatical form" means different things — es = tense + mood; ja/ko = form + politeness; **zh = aspect (了/过/着), no conjugation at all**. **VERIFY FIRST:** do esForEn items expose the infinitive only or the conjugated form? That decides whether this is pure labeling or something deeper.

- **90:** Grammar/conjugation **teaching module ("grammar gym")**. The active-**production** companion to #89's in-quiz transparency: teaches how tenses drive conjugations (regular patterns vs. irregulars/stem-changers) and gives deliberate conjugation practice. **Its own, WALLED-OFF mastery tracker** — same own-lane pattern as #77 (WB lane); it must **never feed the main tracker or the mix-and-match picker**. Two reasons to isolate: (1) keeps the main tracker unfragmented; (2) prevents uneven cross-language depth from manufacturing **false deficits** — a language *without* the module reads as neutral, rather than showing an empty gap next to Spanish (never-punish, applied across languages). **Sibling module** to #36 (listening/speaking) and #62/#83 (writing-system block) — that precedent is what makes it consistent with the §5 decision, not a reversal of it. **SCOPE to Spanish/Romance first**; do not assume it generalizes to the CJK tracks (zh doesn't conjugate). Effort High — a genuinely large standalone module, weight class of #36.

- **92:** Home button next to Back (reported 2026-07-18). **Behavior:** keep the existing Back button; add a **Home** button immediately to its RIGHT that jumps straight to the home page. Show Home **only when the user is 2+ pages deep**; at depth 1 (directly off home) show Back alone — Back already lands on home there, so a Home button would be redundant. **Implementation note:** there's no shared back-button component today — pages roll their own inline "← Back" (e.g. `app/changelog/page.js`, `app/whats-new/page.js`), and some already hardcode `router.push("/")` so their "Back" is effectively "Home." Cleanest path = introduce a shared Back/Home control (or a small header bar) that computes nav depth and conditionally renders Home, then migrate pages onto it. Depth source (a session nav stack/breadcrumb vs `window.history` length) is a build-time decision — a maintained stack is more reliable than history length across deep links/refreshes. Pairs with #85 (top-bar Help) and #84 (dashboard retire), which also touch the home/nav chrome. ADHD fit: fewer taps to get home. Effort Low-Med; Importance Med.

## Internal

- **zh (Mandarin) TTS wired in `generate-tts.mjs`.** Track `zhForEn` added to
  `TRACK_VOICES`, `VOICE_KEYED_TRACKS`, and `LANG_RULES`.
- **Voice: `cmn-CN-Wavenet-A`** (female). No `cmn-CN` Neural2 exists — the family
  is Chirp3-HD (no SSML/rate, unusable), Standard, and Wavenet. Second track after
  ru on Wavenet, same cause; Wavenet honors `<lang>`/`<break>` + rate, required for
  the en→zh production handoff. Locale derives as `cmn-CN` from the voice name via
  the existing `slice(0,2)` split — no parsing change.
- **Prompt-shape rules (`LANG_RULES.zh`):**
  - Recognition `'X (pīnyīn)' 是什么意思？` — pinyin stripped via the shared
    `stripCJKReadingParens` (hanzi-preceded paren = reading aid); headword spoken
    as-is (`knownQuoted`).
  - Production `'X' 用中文怎么说？` — English gloss wrapped in `<lang xml:lang="en-US">`.
    The WB `fvocab` category is production-framed, so ~600 items exercise the handoff.
  - trad `Translate: '...'` → whole-native English (shared branch); zero-hanzi
    prompts → native (`nativeWhenNoTarget`); ` — "hint"` tails → native span.
- **`subReading` deliberately DEFERRED** (not "not needed" like ko/ru): zh is
  logographic, so the isolated-headword 多音字 risk is real, but raw pinyin in
  `<sub>`/`<phoneme>` isn't verified to render correct tones on cmn-CN Wavenet.
  Engine-default readings this pass; heteronyms are the top audition item; a
  curated `<phoneme alphabet="pinyin">` pass is a follow-up only if audition shows
  the engine both mis-reads and honors the tag.
- es/frCa/de/ja/ko/ru rules untouched → **no `--force` needed** on existing tracks.
- Code-only delta: no version files (ledger chat owns `lib/version.js`); no
  dev→main merge. zh rides the batched ru + ko + zh prod release later.

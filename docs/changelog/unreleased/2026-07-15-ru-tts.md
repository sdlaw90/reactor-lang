# ru TTS pass — generator support

## User-facing
None yet. This adds the ru-RU audio-generation path to the TTS pipeline. Clips
are generated to the dev bucket only and stay excluded from progress aggregates
until the listening/audio feature ships. Russian learners get spoken prompts
when the ru track's audio reaches prod (a later ledger release).

## Internal
- `scripts/generate-tts.mjs`: added the `ru` entry to `LANG_RULES` and
  `ruForEn` to `VOICE_KEYED_TRACKS` (voice-keyed from day one).
  - Recognition `'X' значит...` spoken as-is (knownQuoted). Production
    `Как сказать 'X' по-русски?` gets the English gloss in a native `en-US`
    `<lang>` span; greedy `.+` so glosses with inner apostrophes
    (`with one's own eyes`) don't truncate on the closing quote.
  - trad `Translate: '...'` → shared whole-native branch (en-US-Neural2-C).
  - NO subReading (Cyrillic is phonetic — ko's divergence from ja transfers).
  - New `stripCyrillicGlossParens` deriveSpoken drops the one curated gram
    English gloss paren that trails Cyrillic (`Я студент. ("I am a student.")`).
    Production glosses (Latin-preceded) and trad parens are untouched. Strips
    after hashing, so keys stay client-derivable.
  - es/fr/de/ja/ko rules byte-unchanged → those tracks need no `--force`.
- `TRACK_VOICES.ruForEn` is a **deliberately-invalid placeholder**
  (`ru-RU-PIN-FROM-VOICES-LIST`): the voice preflight hard-fails on it and
  prints the live ru-RU voices, so the first real run doubles as `voices:list`.
  PIN the real voice before the paid `--upload` run — do not guess (letters
  churn). Omitting the entry instead would fall back to `es-US-Neural2-A`, which
  exists → preflight passes → Russian synthesized in a Spanish voice; the
  placeholder prevents that failure mode.
- Expected extraction: ~246 clips (58 curated + 188 WB, one question/word),
  fono excluded. One benign review flag: `'Стол' — какого рода?`.
- Dev-only this pass; prod rides `sync-tts` at the ledger release.

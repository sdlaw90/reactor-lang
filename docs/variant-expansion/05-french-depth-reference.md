dedicated `verbo` category so future variant-expansion has a real base._

## Method
Generated by a per-(category × CEFR band A1–C2) workflow (~44 agents), each
generation batch passed through a **native-French adversarial verify** stage
(natural? single correct answer? distractors truly wrong? level right?). Then,
deterministically: **verbecc-verified** every simple-tense verb form (dropped
conjugation mismatches — caught 2 real tense-mislabels and some over-strict
reflexive checks), **deduped** against the existing bank, assigned stable IDs
(`fr-vocab-NNN`, `fr-verb-NNN`, `fr-trad-NNN`), and emitted the tag file.

## Structural change — dedicated `verbo` category
frForEn had no `verbo` (verb drills lived mixed inside `gram`). This pass adds a
`verbo` category ("Verbes"), migrates the existing verb drills + this session's
17 person-variants into it (IDs preserved for round-trip), and leaves `gram` for
genuine non-verb grammar (articles, pronouns, agreement, relative pronouns).
`gameEngine.js` is category-agnostic, so no engine change was needed. Content
spans A1–C2 incl. literary C2 (subjonctif imparfait: fût / eût / vînt / pût).

## Tags
`frForEnTags.js` mirrors `esForEnTags.js` (keyed by normalized prompt text):
themes (#88, the 9-theme catalog), and per-verb tense + person (#89). Wired into
the track via `import { THEMES, tagFor }` + `themes: THEMES, tagFor`.

## Review gate
ALL new content is AI-authored + internally native-verified, shipped to beta per
the standard gate. A **fr-FR native reviewer** signs off for real-prod:
packet `review-packets/fr-depth-review.xlsx`; round-trip data
`generated/fr_depth_generated.json` (IDs join back to the track).
